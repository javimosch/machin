# Benchmark report — naive fib(40)

Generated by [`scripts/bench.sh`](../scripts/bench.sh); regenerate with:

```bash
make bench-report          # or: BENCH_RUNS=10 scripts/bench.sh
```

All three contenders run the **same algorithm** — doubly-recursive
`fib(40)` (= 102334155, ~331M calls) — so this is a compiler-vs-compiler
comparison, not an algorithmic one:

- **MFL** — `machin build examples/bench/fib.mfl` (MFL → C → `cc -O2`)
- **C** — hand-written [`examples/bench/fib.c`](../examples/bench/fib.c), `cc -O2`
- **Rust** — hand-written [`examples/bench/fib.rs`](../examples/bench/fib.rs), `rustc -O`

Best of 5 runs. Time is wall-clock; RSS is peak resident set; size is the
on-disk binary. **Numbers are machine-dependent** — they come from whatever
box ran the script. Reproduce them on yours rather than trusting these.

| Implementation | Time (best of 5) | Peak RSS | Binary size |
|----------------|---------------------|----------|-------------|
| MFL (native, cc -O2) | 0.140s | 1.5 MB | 16 KB |
| hand-written C (cc -O2) | 0.120s | 1.5 MB | 16 KB |
| Rust (rustc -O) | 0.240s | 2.0 MB | 12939 KB |

> MFL lowers to ordinary C, so its native binary lands in the same ballpark as
> hand-written C — the remaining gap is codegen overhead, not interpretation.

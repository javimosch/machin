The source is base64

A program is base64 — one function per line, a blank line between functions. The dense form is the source of truth; the readable text is a projection the machine produces on demand.

func fib(n) {
    if n < 2 { return n }
    return fib(n - 1) + fib(n - 2)

Statically typed, zero annotations, native speed

Types are inferred by unification. The compiler emits C and hands it to cc -O2, so values are unboxed and there is no runtime VM.

The whole Go-flavored core

Composite types

slices []T, structs, maps map[K]V — all unboxed.

Concurrency

go goroutines, channels, a per-goroutine arena that bounds servers.

Closures & generics

capturing lambdas, higher-order functions, implicit generics by monomorphization.

Multiple & named returns

q, r := divmod(a, b), comma-ok, and func f() (q, r).

Variadics

func sum(nums...), called sum(1,2,3) or spread sum(xs...).

JSON over HTTP

sockets, json(x) / parse(s, T{}), string ops & routing.

Build a backend with machweb

A web framework written in MFL itself. Compose it with your app and compile to one native binary — no runtime dependencies.

func main() {
    r := new_router()
    route(r, "GET",  "/",          func(req) { return ok_text("home") })
    route(r, "GET",  "/api/todos", func(req) { return ok_json(json(todos())) })
    route(r, "POST", "/api/echo",  func(req) { return ok_json(req.body) })
    serve_router(48080, r)
}
# compose framework + app, compile, run
machin encode framework/machweb.src app.src > app.mfl
machin run app.mfl

Quick start

git clone https://github.com/javimosch/machin && cd machin
make build                              # → bin/machin (needs Go + a C compiler)
bin/machin run examples/demo.mfl
bin/machin build examples/complex/primes.mfl -o primes && ./primes

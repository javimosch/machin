A backend language shaped for machines.
Minimal syntax, no type annotations, one canonical function per line — built so agents write code in the fewest tokens.

GitHub Spec Changelog

This is a program

A `.mfl` file is plain canonical text — one normalized function per line, a blank line between them. No annotations, no formatting choices, nothing an agent spends tokens on that it doesn't have to. The recursive Fibonacci below is complete and runs:

func fib(n){if n<2{return n}return fib(n-1)+fib(n-2)} func main(){println(fib(10))}

Each line is independently addressable — tooling ships, caches, or rewrites one function without touching the rest — and because it is text, it stays greppable and diffable. (A dense base64 form is available via machin pack for distribution.)

Machine-first by design, not by encoding

Human languages carry ceremony — annotations, boilerplate, formatting rules — that an agent pays for in output tokens on every edit. MFL strips it: types are inferred, the layout is canonical, the surface is minimal. You state intent; agents write the code. Measured, this costs an agent ~2.5× fewer tokens to write and edit than the same logic in a base64-encoded form.

intent→ .mfl (canonical text)→ infer + monomorphize→ C→ cc -O2→ native binary

Native speed, zero annotations

Types are inferred by unification; values compile unboxed. There is no interpreter and no runtime VM.

fib(40), ~331M callstime

The whole Go-flavored core

Everything below is expressed in this canonical form and compiled to native code.

Composite types

slices, structs, maps — all unboxed.

Concurrency

goroutines, channels, and a per-goroutine arena that keeps servers memory-bounded.

Closures & generics

capturing function values and implicit generics by monomorphization.

Returns

multiple, named, comma-ok, and variadic parameters.

JSON over HTTP

sockets, serialize and parse, string ops, routing.

One binary

a backend compiles to a single native executable, no runtime deps.

A concurrent HTTP server — one line

This single line is the entry point of a goroutine-per-connection web server (listen, accept loop, dispatch):

func main() { server := listen(48080) for { conn := accept(server) go handle(conn) } }

The machweb framework — itself written in MFL — adds routing and JSON on top. You compose it with your app and compile:

# the machine composes the framework + your intent into one program
machin encode framework/machweb.src app.src > app.mfl
machin run app.mfl

Run it

git clone https://github.com/javimosch/machin && cd machin
make build                                  # → bin/machin (needs Go + a C compiler)
bin/machin run examples/demo.mfl
bin/machin build examples/complex/primes.mfl -o primes && ./primes

Curious what a line says? machin build <file> --emit-c prints the C the machine generated — an inspection escape hatch, not a step you live in.
